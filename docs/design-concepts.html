Bar Tender · design exploration

Four directions for the workspace

The current window works, but it leans on cards, chips and badges. These four redesigns keep the exact same feature set — library, preview, build receipt, review flow, composer — and rebuild it on Notion’s grammar: pages, hairlines, quiet type, and controls that appear only when needed. All of it inside honest macOS chrome.

4 concepts Static HTML/CSS · system fonts only Light mode shown; every token maps to dark

Ground rules, applied to all four

  • Native chrome, not web chrome. Real traffic lights, 13 px controls, SF-style strokes, a vibrancy sidebar. Nothing that a Mac app wouldn’t ship.
  • Hairlines instead of cards. Separation comes from 1 px rules and whitespace. The only shadows belong to real macOS surfaces — menus and floating panels.
  • Text first, no badges. State is a colored dot plus words (“● Waiting for review”), never a filled pill. Kind, refresh, dates are property rows.
  • Hover-revealed controls. Row actions stay hidden until the pointer arrives. One accent color per screen, reserved for the single primary action.
  • Tools are pages. Identity is an emoji and a name, exactly like a Notion page. The generated source is a plain code block, not a “code card”.
01

Page

The most literal translation: the sidebar becomes a Notion sidebar, and each tool becomes a document. Title, emoji, property rows, then the preview, the review request and the build receipt as ordinary page content. The composer is a quiet bar under the page, not a floating pill.

  • Sidebar rows carry the live value right-aligned in tabular numerals — you can read the whole menu bar state without selecting anything.
  • Badges become properties. Kind, state, refresh interval and approval move into Notion-style label/value rows that invite a click to edit.
  • The review flow is a callout with the zsh source as a gray code block — the only place the window asks for a decision, with the single blue button.
  • Technical details collapse into a toggle, Notion-style, so the receipt reads as one line by default.
  • Inspector stays, but you rarely need it — every common edit happens inline on the page.

02

Command

No persistent sidebar at all. The window is one centered document; everything else lives behind ⌘K. The quick switcher is the library — each row already shows the live menu-bar value, so switching context and checking state are the same gesture.

  • The sidebar becomes a modal habit, like Notion’s quick find: open, type, enter. The window keeps a breadcrumb instead of a source list.
  • Live values in the palette mean you can peek at “24:59” or “main ±3” without ever leaving the current tool.
  • Maximum focus for the review flow — when a generated tool needs approval, nothing competes with the code block.
  • Search field sits in the toolbar, the one place Mac users already expect it (Notes, System Settings).
  • Trades away: ambient visibility. You no longer see all tools at once — the menu bar itself has to carry that job.

03

Columns

The most traditional Mac layout — source list, tool list, detail — but executed with Notion discipline. The inspector is gone: settings are just rows at the bottom of the detail pane, and Preview / Source / Activity are underline tabs, like a Notion inline database view.

  • Three densities of the same data. Sidebar = names, list = name + one live line, detail = everything. No row repeats chrome it doesn’t need.
  • Selection is gray, not blue. Only the primary button earns the accent color; lists highlight with Notion’s neutral wash.
  • Tabs replace the inspector. Generated source, approval and runtime state are one click away instead of a second window of form controls.
  • Timer controls are plain bordered buttons in a labeled row — native, reachable by keyboard, invisible when the tool isn’t a timer.
  • Best for a large library. This is the direction that still feels calm with forty tools installed.

04

Menu Bar First

The app is a menu-bar app — so this direction makes the panel the product. One floating surface under the menu bar holds the composer and the whole library; each row shows exactly what that item renders in the menu bar, and expands inline for details. The big window becomes optional.

  • The panel is the app. Compose, review, approve, pause a timer, read logs — all without opening a window. ⌃⌘B toggles it from anywhere.
  • Rows expand like Notion toggles. Port 3000 is shown open: state line, then the two actions that matter — no drill-in, no inspector.
  • Right-aligned values mirror the menu bar verbatim (“♪ Midnight City”, “24:59”), so the panel reads as a legend for the real thing above it.
  • The strip on top is the actual menu bar — the preview stops being a screenshot and becomes the truth.
  • Trades away: long source review is cramped at 372 pt, so “Review in window” remains one click for the big reads.
